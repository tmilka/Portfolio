Vision & Zielsetzung

Ein Portfolio ist mehr als nur eine Visitenkarte – es ist das digitale Äquivalent eines Labors, in dem Theorie auf Praxis trifft. Tim's Space wurde mit dem Anspruch entwickelt, eine Brücke zwischen technischer Präzision und kreativem Design zu schlagen. Während viele moderne Websites auf überladene Frameworks setzen, basiert dieses Projekt auf der Philosophie des Lean Web Designs: Maximale Performance durch minimalen Overhead.

Der Technische Stack

Die Seite ist als Static Site konzipiert, was extreme Ladegeschwindigkeiten und hohe Sicherheit garantiert.

1. Hosting & Infrastruktur

Das Hosting erfolgt über GitHub Pages. Dies ermöglicht nicht nur eine kostenfreie Bereitstellung, sondern integriert den gesamten Entwicklungsprozess direkt in die Versionsverwaltung. Jeder `git push` löst automatisch einen Deployment-Workflow aus.

2. CSS Architektur & Designsystem

Anstatt Frameworks wie Bootstrap zu nutzen, wurde das gesamte CSS von Grund auf selbst geschrieben. Dabei kommen moderne Techniken zum Einsatz:

  • Custom Properties (Variablen): Ein zentrales Farbschema in der `base.css` erlaubt es, das gesamte Theme (Dark-Mode, Akzentfarben) an einer einzigen Stelle zu steuern.
  • Flexbox & Grid: Das Layout ist voll-responsiv. Besonders das Problem des "Sticky Footers" bei wenig Inhalt wurde durch eine intelligente Flex-Struktur gelöst.
  • Glassmorphism: Durch den Einsatz von `backdrop-filter: blur()` und semi-transparenten Hintergründen entsteht ein moderner, futuristischer Look, der die Tiefe des Weltraum-Themes betont.

Interaktivität & Experience

Um der Seite Leben einzuhauchen, wurden gezielt JavaScript-Komponenten implementiert, die die User Experience (UX) verbessern, ohne die Ladezeit negativ zu beeinflussen:

Dynamic Particle Background

Ein HTML5 Canvas-System, das im Hintergrund flüssige Sternen-Animationen berechnet.

Typewriter Engine

Ein custom Script auf der Startseite, das Begrüßungstexte dynamisch generiert.

Workflow & CI/CD

Die Entwicklung folgt professionellen Standards. Änderungen werden in lokalen Feature-Branches entwickelt und getestet, bevor sie via Pull Request in den Main-Branch gemerged werden.
